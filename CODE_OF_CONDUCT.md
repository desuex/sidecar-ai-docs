Code of Conduct

1. Purpose

Sidecar AI Code Documentation is a technical project built around determinism, precision, and structural integrity.

We expect contributors to maintain the same standards in collaboration:
	•	Respect
	•	Clarity
	•	Professionalism
	•	Accountability

This Code of Conduct applies to:
	•	GitHub issues
	•	Pull requests
	•	Discussions
	•	Community chats
	•	Design debates
	•	Any public project space

⸻

2. Our Standards

Participants are expected to:
	•	Use respectful language
	•	Critique ideas, not people
	•	Provide constructive feedback
	•	Back claims with reasoning
	•	Stay on topic
	•	Assume good intent
	•	Accept technical disagreement professionally
	•	Respect differing levels of experience

Technical rigor does not excuse hostility.

⸻

3. Unacceptable Behavior

The following behaviors are not tolerated:
	•	Personal attacks
	•	Harassment or intimidation
	•	Dismissive or insulting language
	•	Discriminatory remarks
	•	Threats of any kind
	•	Derailing discussions intentionally
	•	Repeatedly ignoring contribution guidelines
	•	Public shaming of contributors
	•	Publishing private information without consent

This project prioritizes technical clarity over ego.

⸻

4. Technical Disagreements

This project encourages strong technical debate.

Allowed:
	•	Critiquing architecture
	•	Challenging design decisions
	•	Proposing alternatives
	•	Rejecting proposals with reason

Not allowed:
	•	Attacking the proposer
	•	Dismissing without explanation
	•	Gatekeeping participation

Arguments must focus on:
	•	Determinism
	•	Safety
	•	Performance
	•	Security
	•	Correctness

⸻

5. Decision-Making

Maintainers have final authority over:
	•	Architecture
	•	Roadmap
	•	Breaking changes
	•	Security posture

Disagreement is acceptable.

Disrespect is not.

⸻

6. Reporting Violations

If you experience or witness unacceptable behavior:
	•	Email: conduct@sidecar-ai.dev
	•	Provide details and evidence if possible

Reports will be handled confidentially.

⸻

7. Enforcement

Project maintainers may:
	•	Request clarification
	•	Issue warnings
	•	Remove comments
	•	Lock discussions
	•	Reject contributions
	•	Suspend or ban contributors

Enforcement decisions are final.

⸻

8. Scope

This Code of Conduct applies within project spaces.

Private conduct outside project context is not policed unless it directly affects project safety.

⸻

9. Professionalism Standard

This project operates under professional engineering standards.

We value:
	•	Precision
	•	Accountability
	•	Evidence-based reasoning
	•	Clear communication
	•	Long-term maintainability

We do not tolerate chaos-driven collaboration.

⸻

10. Attribution

This Code of Conduct is inspired by common open source governance practices but adapted to the project’s architectural philosophy.

⸻

By participating, you agree to uphold these standards.
